# G-065 risk-tiered quality-gate audit

Date: 2026-07-14

## Baseline and decision

Before G-065, `.github/workflows/fresh-clone-dev.yml` was path-filtered and
owned only setup portability. The tag publish workflow ran licensing and
desktop tests before publication. No workflow ran the 415-file default suite,
all-source coverage, TypeScript, test-audit topology, or real runtime graph on
general pull requests. Strong G-064/G-066/G-068 local controls were advisory.

G-065 adds one dependency-free Node runner and a reusable GitHub Actions
workflow. Pull requests always run the base contract; changed paths can only add
costly boundary controls. Merge/release profiles run all controls. The publish
job has `needs: quality`, so npm/GitHub publication cannot start after a failed
or absent release-profile result.

## Contract matrix

| Boundary | PR base | PR mapped path | Merge/release | Separate topology |
|---|---:|---:|---:|---:|
| TypeScript | required | required | required | — |
| default suite under all-source coverage | required | required | required | — |
| coverage denominator/risk ratchets | required | required | required | — |
| test audit + deterministic static guards | required | required | required | — |
| harness safety | — | config/harness paths | required | — |
| real runtime graph | required | required | required | — |
| mutation strength | — | protected chokepoints/policy | required | — |
| Pack compatibility | — | Pack contract/policy | required | — |
| CLI build | — | — | required | rebuilt in publish runner |
| fresh-clone Node/npm matrix | — | existing path filter | existing | macOS/Windows workflow |
| packaged `npx` customer smoke | — | — | after shared gate | tag publish runner |

The base workflow itself has no `paths`/`paths-ignore` filter. This avoids the
required-check pending/bypass ambiguity and GitHub's 300-file path-filter
evaluation limit. `merge_group` is present so a future merge queue receives the
same required check. Checkout does not persist credentials, job permissions are
read-only contents, and no secret/provider input is passed to pull-request code.

## Coverage ratchet

The gate derives 965 eligible production sources from the repository and
requires all 965 in the V8 summary. It enforces exact G-068-era no-regression
floor fractions on eleven Tier-0/Tier-1 surfaces while reporting Tier-2 Components and
the CLI exception without presenting their low percentages as adequate. The
CLI remains protected by fresh-clone and packaged production smokes.

Fresh coverage evidence:

- 415/415 Vitest files passed; 3,245 passed plus one intentional conditional
  skip (3,246 collected)
- lines 39.71% (13,660/34,396), branches 33.83% (9,196/27,179), functions
  37.28% (2,795/7,497), statements 39.12% (14,591/37,297)
- 965/965 eligible files reported; 432 files have zero covered lines
- every committed risk floor passed
- executable audit schema 4 reports the quality workflow as the repository's
  full-suite reference

These numbers are diagnostics and ratchets. Named boundary receipts remain
mandatory where line coverage cannot prove behavior.

## Deliberate controls

`scripts/quality-gate.test.mjs` proves:

- ordinary PRs cannot omit the always-on contract
- runtime smoke is always-on, including for transitive dependencies outside the
  runtime source tree, while protected workflow paths add mutation controls
- quality-policy changes conservatively add every conditional lane
- Pack manifests add compatibility checks without unrelated conditional work
- merge/release runs all conditional and release-only lanes
- unknown profiles, empty PR diffs, absolute/traversal paths, and unknown CLI
  switches fail closed
- non-zero exit, signal, launch error, and zero-exit/missing semantic evidence are
  named failures
- eligible-source removal, a lowered risk floor, and a one-uncovered-line
  regression hidden by two-decimal rounding fail independently
- workflow triggers, read-only permission, stable job name, absent path filter,
  non-persisted checkout credentials, release dependency, and preserved
  fresh-clone/packaged-smoke boundaries are executable assertions
- always-on package scripts cannot introduce opportunistic `npx` downloads;
  design-token validation is dependency-free `.mjs` compatible with Node 20+

The suite passes 17/17 with no skip.

## Measured release-profile replay

The exact `npm run quality:gate -- --profile release` command passed all 18
lanes on Node 22.18.0/npm 11.6.0 in 98,825 ms. Largest lanes were default
coverage (46,366 ms), real runtime graph (32,502 ms), mutation strength (8,635
ms), and TypeScript (1,881 ms). This is 13.7% of the measured 12-minute local budget;
no sharding, new cache, dependency, service, credential, or billable provider
was added.

The local budget is evaluated after each lane completes and is not presented as
a hard process-tree timeout. The hosted workflow's 15-minute job timeout is the
hard ceiling; the runtime and mutation controls retain their own bounded cleanup
and the release replay confirmed no disposable processes or directories remained.

The runtime receipt completed task `5bdbb6cc-ca98-4416-8a94-48cd9bead234`,
workflow `71661f36-1cb3-43d0-9bd2-7cf37cd01d96`, workflow child
`5d004ee5-139c-4b7b-8b4b-473db5be3bed`, and conversation
`b3093650-d2a0-4795-8eaa-54e0e1cfe45f` with `stream.completed`. Mutation
evidence killed 7/7 required faults, retained the intentional survivor, matched
the 63-test restoration matrix, and removed its disposable copy.

## Live required-check disposition

Read-only GitHub API inspection returned `Branch not protected` for `main` and
an empty repository-ruleset list. The workflow will run on every pull request,
but GitHub does not yet prevent an authorized user from merging around it. The
intended rule is a strict required `Relay quality gate` check. Enabling that rule
would materially change Relay's current direct-main workflow and is therefore
an explicit external operator gate, not a silent repository mutation.
G-065 closes with the workflow implemented but live merge enforcement pending;
the check is not described as merge-required until that external setting exists.

## Rollback

Revert the G-065 commit to restore the prior workflow topology. Fresh-clone and
customer-identical `npx` smoke implementations are not removed. If the live
required check is enabled later, remove or retarget it before reverting the
workflow so pull requests do not remain pending.
