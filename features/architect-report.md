# Architect Report

## Change Impact Analysis — G-095 Host fulfillment contract

### Proposed change

Extend the signed offline `orionfold.license/v1` payload with a versioned Host
grant while keeping Pack rights, OCI authenticity, and registry access separate.

### Blast radius

| Layer | Surfaces | Impact |
|---|---|---|
| License contract | canonical payload, verifier, fixtures | additive signed grant and named policy |
| Host domain | future G-083 admission/lifecycle | one reusable source of capacity/lapse truth |
| Distribution | npm Host / OCI Cell boundary | no artifact change; authority is clarified |
| Public docs | README and trust/Host guides | Core/Packs/Host/bundle become independently legible |
| Website coordination | G-030 issuer/catalog handoff | exact schema/vectors replace inferred semantics |

**Classification:** High — cross-repository commercial and security contract,
but deliberately no database, runtime-registry, or live-commerce mutation.

### Dependency trace

```text
Website catalog + issuer
  → signed orionfold.license/v1 envelope
      → Relay signature verification
          → Host grant parser
              → G-083 admission and lifecycle policy

G-094 release manifest + OCI signature
  → artifact authenticity (independent; never a license claim)
```

### Migration requirements

- Database/bootstrap/migration: none; the canonical store retains exact files.
- Outer license schema: remains v1; `grants` is an additive signed extension.
- Existing Pack licenses: unchanged and never parsed as Host grants.
- API versioning: none in G-095; G-083 consumes the module internally.
- TDR: amend TDR-044's provisional entitlement and distribution consequences.

### Risks and mitigations

- **Signature confusion:** verify raw payload before Zod parsing or defaults.
- **Authority confusion:** forbid registry tokens in licenses and keep OCI
  signature errors outside the license error family.
- **Offline copying:** describe `hosts` as a signed commercial/local claim, not
  globally perfect DRM; no hidden activation service.
- **Lapse damage:** encode continuity actions separately from paid expansion.
- **Capacity residue:** G-083 must call admission before every allocation; the
  G-095 pure matrix becomes its normative test oracle.
- **Issuer drift:** commit schema and canonical dev-signed vectors that Website
  can reproduce byte-for-byte.
- **Commercial drift:** Relay never embeds amounts; Website pricing JSON remains
  the amount/term authority.

### Recommended approach

Proceed additively. Accept `product:relay-host`, a per-product grant, explicit
one-Host/ten-Cell launch shape, separate Pack rights, customer-protective lapse,
same-licensee replacement, and indefinite critical-security eligibility for
existing compatible Cells. Do not build Host effects or Website commerce until
their owning goals consume this contract.
